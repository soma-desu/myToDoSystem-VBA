# Function Specification

1. To Doの基本管理機能
    - 通常のTo Do
        (a) 新規To Doの追加  
        (b) 既存To Doの編集  
        (c) To Doの削除  
        (d) To Doの実行時間/休憩時間の設定
    - リピートのTo Do
        (a) 新規To Doの追加  
        (b) 既存To Doの編集  
        (c) To Doの削除  
        (d) To Doの実行時間/休憩時間の設定

2. 通常To Doの設計
    - 実体
        単独で存在
    - プロパティ
        - ID
        - ToDoの名前
        - ToDoのタイプ
        - 実行予定日
        - 実行予定時間
        - 締切日時
        - SubPjのID
        - 推定所要時間
        - 休憩予定時間
        - 開始時間
        - 終了時間
        - 優先度
        - 生成日時

2. リピートTo Doの追加・管理機能
   (a) リピート基準の登録  
       以下の条件でリピート基準を設定できます：  
       ・毎週の曜日指定  
       ・毎日（土日祝日の除外設定も可能）  
       ・隔週での曜日指定  
       ・毎月の特定の日（日にち指定）  
       ・毎月の最初の平日  
   (b) タスクの自動生成  
       登録したリピート開始日と終了日に応じて、自動的に各日程のタスクを作成します。  
   (c) 柔軟な編集機能  
       個別のインスタンス（特定の日）のみの編集と、リピート全体への一括反映を選択して実行できます。  
       例えば、ある日の実行予定時間を変更した際、その変更をその日だけに適用するか、他の全日程にも反映させるかを選択可能です。  

3. 2週間分のカレンダー表示機能（視覚的確認）
   2週間分のタスクを横並びで表示します。  
   (a) 縦軸に朝6時から深夜24時まで、5分刻みの時間軸を表示します。  
   (b) 実行予定時間を視覚的に確認できるレイアウトにします。  

4. 実行ログの保存と振り返り機能
   (a) 実行したタスクのデータを保存します。  
   (b) 特定の日に「どのタスクにどれくらいの時間をかけたか」を後から振り返り、分析できるようにします。
